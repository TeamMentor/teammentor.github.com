Consuming TeamMentor Data

TeamMentor was designed from the ground up to facilitate the easy access and manipulation of its data.

There are number of ways do it by:

  • Accessing the Webservices directly
  • Using the webapp Control Panel WebServices GUI
  • Using the webapp Javascript API (which wraps the WebServices)
  • Directly consuming the server side DLLs (using or nor the WebServices methods)
  • Directly manipulating the XML files (either at the master XML Git repository or at a local install/fork/clone)
  • Using the O2 Platform and its multiple TeamMentor data manipulation and viewing GUIs

Accessing the Webservices directly

Although all data traffic goes though the WebServices layer, there is currenlty only one .NET WebService *.asmx file. This makes it easy to consume and centralizes all exposed methods in one location.

This file can be found at the  /aspx_pages/TM_WebServices.asmx .For example on the current test servers you will find it at:

  • http://50.18.82.146:8081/aspx_pages/TM_WebServices.asmx  (OWASP Library)
  • http://50.18.82.146:8082/aspx_pages/TM_WebServices.asmx  (SI Library)

To get the WSDL from this webservice just append ?wsdl to the url (i.e. /aspx_pages/TM_WebServices.asmx?wsdl)
